Built for the real bottleneck

The job is to publish clearly, not run a CMS fleet.

TinyBlog narrows the stack to a single PHP app, a SQLite database, and one embed script. Authors get a minimal admin writing surface; readers get static-style canonical pages, RSS, and an accessible widget that can live inside any existing site.

What the tool actually does

Three ways TinyBlog helps before the next deploy.

01

Render blog feeds anywhere

Drop in the script, set an endpoint, and render feed, post, or subscribe widgets with CSS-variable overrides.

02

Keep publishing local to your host

Posts, uploads, subscribers, settings, RSS, and sitemap data stay in your PHP hosting account by default.

03

Audit the security path

Prepared SQL, escaped templates, CSRF, session timeout, rate limits, upload whitelists, and CORS checks are intentionally visible.

How it flows

One path from upload to embedded update.

01

Deploy

Upload the files, enable PHP 8 with PDO SQLite, and let the app create data/tinyblog.db.

02

Write

Create the first admin, publish Markdown posts, upload safe images, and set allowed widget origins.

03

Embed

Paste a one-line script into any allowed site and fetch only the JSON that the widget needs.

Quick start

Get from cPanel to a live feed fast.

<script
  src="https://blog.example.com/tinyblog-widget.js"
  data-tb-config='{"site":"store-1","endpoint":"https://blog.example.com/api","widgetType":"feed"}'>
</script>
