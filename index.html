A self-hosted blog in one script tag.

TinyBlog is a single PHP file with a SQLite database and a one-line JavaScript embed. Render a feed, a single post, or a subscribe box on any site — publishing and data stay on your host, not a vendor cloud.

Read the docs View on GitHub
store-page.html 
<script
  src="https://blog.example.com/tinyblog-widget.js"
  data-tb-config='{"site":"store-1",
    "endpoint":"https://blog.example.com/api",
    "widgetType":"feed","maxItems":5}'>
</script>

// That's the whole client-side integration.
// Posts render from JSON your PHP host serves —
// prepared SQL, CORS allowlist, no third parties.

SYS · 01

The whole system, on one sheet.

No CMS fleet, no vendor cloud. One PHP app writes to SQLite, renders canonical pages and RSS, and serves the JSON the embed reads.

Node 01

Any site

One <script> tag renders feed, post, or subscribe. Zero dependencies, no build step.

Node 02

Your PHP host

tinyblog.php + SQLite: admin, posts, uploads, subscribers, settings. Data never leaves your account.

Node 03

Reader surface

Canonical post pages, tag & search, RSS and sitemap, plus a CORS-allowlisted JSON API for the widget.

SYS · 02

From cPanel to a live feed in minutes.

Upload one folder, enable PHP 8, and the app creates its own database. Then write and embed.

01 / Deploy

Upload & go

Drop the files on any PHP 8 host with PDO SQLite. The app creates data/tinyblog.db on first run — no migrations to run by hand.

02 / Write

Publish in Markdown

Create the first admin, write Markdown posts, upload image-only media, and set your allowed widget origins.

03 / Embed

One tag, any page

Paste the script on an allowed site. It fetches only the JSON the widget needs and renders it inline.

04 / Audit

Read the whole path

Prepared SQL, escaped output, CSRF, rate limits, upload whitelists and CORS checks stay visible in one readable file.

SYS · 05

What it deliberately doesn't do.

Honest scope beats an inflated feature list. When you outgrow these limits, the README's "How to scale" section points the way.

Not a CMS fleet

No multi-tenant admin, no page builder, no theme marketplace, no plugin API. One app, one blog, one readable file.

Not a heavy stack

No Node runtime, no build step, no Composer dependencies, no background job queue. SQLite until you genuinely outgrow it.
